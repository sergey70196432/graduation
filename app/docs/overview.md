# Обзор и возможности приложения

Это учебное мобильное приложение (MVP) для дипломного проекта:

- **камера** (Android/iOS) → поток кадров
- **офлайн‑инференс** YOLO в формате **TFLite**
- **отрисовка боксов** + label + confidence
- **колонка “Знаки”** со списком найденных знаков (SVG) и “липкостью” (debounce) отображения
- **выбор модели** из списка (скачивание по сети и кэширование в `DocumentDirectory`)
- **video‑режим**: выбрать видео из Files/Photos и гонять инференс по кадрам

## Ключевые принципы

- **Офлайн‑first**: после скачивания модель и `labels.txt` лежат локально и работают без интернета.
- **Один код для iOS и Android**.
- **Скорость/простота**: минимальная архитектура (hooks + несколько утилит), без тяжёлых state‑машин.

## Что умеет (в текущем состоянии)

- **Camera mode**
  - превью камеры
  - детекция в реальном времени (frame processor)
  - overlay (bbox) в debug сборке всегда, в релизе — скрыто (подробнее в `docs/build-modes.md`)
  - выбор модели (попап в колонке “Знаки”)
  - экспорт логов производительности (если включено флагом, см. `docs/performance.md`)

- **Video mode**
  - выбор видео из Files или Photos
  - проигрывание + авто‑инференс на таймкоде
  - bbox overlay поверх кадра

## Ограничения

- **Тестирование**: на данный момент приложение запускалось и тестировалось **только на iOS**. Android‑часть может требовать дополнительной настройки/фиксов (Gradle/JDK/NDK, совместимость native‑модулей).
- В приложении предполагаются TFLite‑модели, у которых **NMS “вшит” в модель** и выход обычно имеет форму **`[1, N, 6]`**:
  - `x1, y1, x2, y2, score, classId`
  - `N` обычно ограничено (например 300)

Если экспортировать модель без встроенного NMS, потребуется другой декодер (в этой ветке проекта он не поддерживается).

## Состояние “нет выбранной модели”

В приложении нет встроенной модели. При первом запуске (или после “Очистить хранилище”) может быть состояние,
когда активная модель не выбрана — в этом случае поверх превью камеры показывается затемняющий оверлей
с сообщением “Нет выбранной модели”.
