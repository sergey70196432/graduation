# Экраны и UX

В приложении два режима:

- **Camera** — реальное время с камеры
- **Video** — тест на заранее записанном видео

Переключение делается в `App.tsx` (простое состояние `mode`).

## CameraDetectionScreen

Файл: `src/screens/CameraDetectionScreen.tsx`

### Колонка “Знаки”

- показывает найденные знаки (SVG) и метаданные
- “липкость” (debounce): если знак пропал из детекций, он остаётся в списке ещё некоторое время (сейчас 3 секунды)
- кнопка **“Модель”** открывает попап выбора модели

### Попап “Выбор модели”

- “Обновить” — скачивает манифест моделей
- “Скачать” — скачивает `model.tflite` и `labels.txt` в `DocumentDirectory`
- “Выбрать” — делает модель активной
- “Удалить” — удаляет скачанную модель с устройства (обычно доступно только если модель не активная)
- “Очистить хранилище” — удаляет **все** скачанные модели и `labels.txt` с устройства (полная очистка каталога моделей)

Пока идёт очистка, показывается спиннер и кнопки в попапе блокируются. После завершения список обновляется.

Примечание по iOS: попап реализован **не через `Modal`**, а как overlay‑`View` внутри экрана (чтобы избежать падений по ориентациям/VC).

### Debug UI

- В debug‑сборке (`__DEV__`) bbox overlay рисуется всегда
- Кнопка “Debug” присутствует только в debug‑сборке

Подробности см. `docs/build-modes.md`.

### Классификация скоростных знаков

В `Camera` режиме есть дополнительная стадия: **speed classifier** — отдельная TFLite модель, которая уточняет значение скорости на знаках (пишет `refinedLabel/refinedConfidence`).

Подробности: `docs/speed-classifier.md`.

## VideoDetectionScreen

Файл: `src/screens/VideoDetectionScreen.tsx`

### Выбор видео

Есть два источника:

- **Files** (через `react-native-document-picker`)
- **Photos** (через `react-native-image-picker`)

На iOS некоторые видео из iCloud/Files могут быть недоступны напрямую, поэтому экран старается копировать файл в sandbox (cache).

### Инференс по кадрам

- берём thumbnail кадра (`react-native-create-thumbnail`) на текущем таймкоде
- декодируем JPEG
- делаем letterbox на `inputSize`
- запускаем TFLite
- декодируем выход `[x1,y1,x2,y2,score,classId]` и рисуем overlay

## Авто‑инференс

Авто‑инференс **всегда включён**: отключить его нельзя. Экран работает как “processing loop”.

Как это устроено (упрощённо):

- как только закончилась обработка и мы получили результат детекции, **сразу** планируем следующий прогон
- берётся текущий таймкод:
  - если видео **играет** → `videoPositionMs`
  - если **на паузе** → `timestampMs` (ручной таймкод)
- если предыдущий прогон ещё идёт — новый запуск не стартует (защита от параллельной обработки)
- защита от “молотилки”: если таймкод не меняется (например пауза), делаем небольшую задержку, чтобы не крутить один и тот же кадр в бесконечном цикле

Раньше тут был “регулируемый FPS” (ограничение частоты прогонов). Теперь его нет: скорость обновления зависит от:

- скорости `createThumbnail`
- скорости декодирования JPEG
- скорости TFLite инференса
- скорости декодирования bbox
