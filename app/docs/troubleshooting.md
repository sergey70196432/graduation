# Troubleshooting (частые проблемы)

**Примечание по платформам:** приложение запускалось и тестировалось **только на iOS**. Если проблема относится к Android, возможно, это “неподдержанный” сценарий в текущем состоянии проекта.

## Модель не загружается (iOS)

### Симптом

- в UI: “Модель не загрузилась”
- в логах: `install() ... false` или похожие сообщения из `react-native-fast-tflite`

### Причина

На RN 0.84 iOS часто работает в режиме bridgeless, из-за чего некоторые реализации `install()` ломаются без патча.

### Решение

- убедиться, что после `npm install` применились патчи (`patch-package`)
- переустановить pods
- в Xcode: **Clean Build Folder**

## Ошибки сборки Android (node/npx не найден)

Если Gradle не видит `node`, проверь переменные окружения и то, как запускаешь Android Studio (часто нужно запускать IDE из терминала, чтобы она унаследовала env).

## Video screen: iOS “Cannot Open” (-11829)

Причины:

- файл не скачан локально (iCloud)
- неподдерживаемый кодек/контейнер

Решения:

- использовать MP4 (H.264)
- выбрать видео из Photos
- дать приложению доступ к файлам и убедиться, что файл локальный

## UI перекрывается Dynamic Island / notch

Решение: использовать `react-native-safe-area-context` и учитывать `insets` (особенно в landscape).

## SVG знаки не импортируются / Metro не видит `react`

Если SVG лежат в `shared/`, Metro должен уметь резолвить зависимости из `app/node_modules`.
Проверь `metro.config.js` (watchFolders/extraNodeModules).
